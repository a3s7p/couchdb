Connecting to CouchDB

By default, raw CouchDB access is via the sub-directory. I.e.:

If you would rather access CouchDB via the naked domain (i.e. without the '/couchdb' URL suffix) then log into your server via ssh and enable the alternate Nginx config we supply:

rm /etc/nginx/sites-enabled/couchdb
ln -s /etc/nginx/sites-available/couchdb-standalone /etc/nginx/sites-enabled/couchdb-standalone
systemctl restart nginx

Note: This page will no longer be available. To connect to Fauxton, add the '/_utils/' suffix to the domain.
